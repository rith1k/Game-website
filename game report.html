Website Report

My Game Website: Hidden Object

The game of my selection is a hidden object game . The motive of the game is for having a list of items blended with a background image,
and to allow a player to guess all the items before the time runs out, and get the highest score possible.

Local Storage

The entire game uses the local storage as its main form of storage. All registered users have their information stored in local storage, and the logged-in details are stored in the session storage.
The local storage stores the highest score for each user, for each level played and keeps the default score as undefined.

Game functions:

Game Levels:

The game has 2 levels. Each level has 60 seconds of time, and the user has to guess the maximum amount of items before the time runs out. Each level has a capped amount of lives, 5 for level 1, and 3 for level 2. The game ends if the user loses all his lives.

Objects and arrays used:

The whole game, and the website uses a bunch of objects and arrays which help in inserting and retrieving information from and to the local storage.

Advanced gameplay:

Function check answer

This function runs every time a player makes a guess and the right answer increases the score of the user and the wrong guess decreases a life for the user's current game.

Function move () :

Every time the player guesses the right answer, the function move is initiated and an animation is applied and the item moves to its designated spot in the item bar.

Countdown timer function:

Each level has a capped timer of 60 seconds. Either the user has to guess the answers or the time runs out. The timer resets if the user runs out of time, or if he loses all of his lives.

Animations and graphics used:

The game has several graphic elements,each level has different objects blended in with the background images, and also a panel of the same objects but with filtered brightness.
Animation have been applied all through the website, but in the game, animations are brought in when a player makes a guess.

User Account

Registration

The register function takes in all the values inputted by the user in the username, password, password confirm and email text fields, and verifies if the password and password_confirm match, and subsequently,
registers the user into the database.

Login

The login function essentially checks if the username and the password entered in the form matches with the registered user's data. If the username doesn't match, the user is prompted to create an account.

Logout

The logout function essentially checks if the user is logged in or not, if the user is logged in,the register and login forms disappear, and a logout button is shown if in case the user wants to log out.
The logout button only shows up if a user has successfully registered, and logged in with the correct information.

Leader Board

Every time a user plays the game, his new score is checked with his previous highscore, and if its higher, the score gets updated. But if the user has no previous score, his first score is taken as his score and displayed on the leaderboard.

The leaderboard table creates a new row every time a new user plays any of the 2 levels. The leaderboard only stores the values of registered users.

Sorting

The table can be sorted by any column the user wishes for, but by default, the table is automatically sorted from the highest to the lowest of scores for level 1. The sorting algorithm is a 3rd party library inherited in the main program.

Third Party library

The Third library used in this website was a sorting library called sortable.js, that specifically helps in sorting a table by any column, in ascending or descending order.
Link for the third-party library: https://kryogenix.org/code/browser/sorttable/
